# Security Advisory – SA-2025-001
**Corsair iCUE 5 Configuration File Write Loop Leading to Resource Exhaustion and CPUID Deadlock**

**Release Date:** 2025-12-05
**Last Updated:** 2025-12-05,
**Severity:** High  
**Status:** Unpatched  
**Affected Product:** Corsair iCUE 5.x  
**CVE Identifier:** CVE-TEMP-iCUE-2025-LINKED (Pending)

---

## Summary
Corsair iCUE 5 contains a defect in the backend configuration file event handler that triggers constant high-frequency writes to `config.cuecfg` and associated log files.  
This creates:

- massive disk I/O  
- accelerated SSD/NVMe wear  
- persistent file lock contention  
- deadlock of the Corsair CPUID Service

The CPUID service, unable to read configuration data due to the lock, becomes stuck and fails to deliver sensor, fan, and pump telemetry.  
This causes cooling curves and temperature mappings to fail system-wide.

---

## Affected Versions
All tested iCUE 5.x releases:

- 5.10.x  
- 5.9.x  
- 5.8.x  

**iCUE 4.x is not affected.**

---

## Impact

### 1. Resource Exhaustion
- ~1 GB/hour write activity  
- SSD/NVMe endurance degradation  
- Increased system latency  

### 2. Service Deadlock
- CPUID stuck in **Starting**  
- Loss of fan & pump control  
- Incorrect CPU/GPU temperature mapping  
- Cooling curves not applied  

### 3. Operational Degradation
- Failed profile loading  
- Missing sensors  
- Broken telemetry  
- No rollback path available  

---

## Technical Details
A regression in the file system watcher subsystem causes iCUE to interpret normal read activity as a write event.  
This triggers an endless loop:

1. CreateFile  
2. QueryOpen  
3. Write  
4. CloseFile  
5. Repeat hundreds of times per second

This blocks dependent components (like CPUID) from reading the file, resulting in a startup deadlock.

---

## Reproduction Steps
1. Install iCUE 5.x  
2. Allow startup  
3. Open Task Manager or Resource Monitor  
4. Observe continuous writes from `iCUE.exe`  
5. Restart CPUID service  
6. Service hangs indefinitely  

---

## Mitigation (Temporary Only)
- Disable all optional logging  
- Limit iCUE startup modules  
- Use third party fan control software  
- Monitor temperatures manually  

No official fix exists.

---

## Vendor Status
Corsair has acknowledged user reports but has not provided:

- a development timeline  
- technical analysis  
- rollback mechanism  
- archived installers
