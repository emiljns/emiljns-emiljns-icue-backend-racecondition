# Technical Advisory TA-2025-001
Corsair iCUE 5 Configuration File Write Loop Causing Resource Amplification and CPUID Service Deadlock

Release Date: 2025-12-05  
Last Updated: 2025-12-05  
Operational Impact: High  
Security Impact: Not Demonstrated  
Status: Unpatched  
Affected Product: Corsair iCUE 5.x  

---

## Summary

Corsair iCUE 5 contains a regression in its configuration file event handling subsystem that results in continuous high frequency write activity to `config.cuecfg` and associated log files.

Under normal operation, this behavior produces sustained disk I O of approximately 1 GB per hour, leading to:

- Accelerated SSD or NVMe endurance consumption  
- Persistent file lock contention  
- Service startup instability  
- Deadlock of the Corsair CPUID Service  

When the CPUID service attempts to access configuration data during initialization, it may become indefinitely stuck due to file lock contention. This prevents proper telemetry delivery and can result in loss of sensor, fan, and pump control functionality.

No security boundary violation or privilege escalation has been demonstrated. The impact is limited to system reliability, resource exhaustion, and service availability.

---

## Affected Versions

All tested iCUE 5.x releases:

- 5.10.x  
- 5.9.x  
- 5.8.x  

iCUE 4.x is not affected.

---

## Impact

### Resource Amplification

- Approximately 1 GB per hour sustained write activity  
- Increased system latency  
- Accelerated SSD or NVMe endurance degradation  
- Elevated background disk activity during idle conditions  

### Service Deadlock

- CPUID service stuck in Starting state  
- Loss of fan and pump telemetry  
- Incorrect or missing temperature mappings  
- Cooling curves not applied  

### Operational Degradation

- Failed profile loading  
- Missing sensors  
- Broken telemetry  
- No official rollback mechanism  

---

## Technical Details

A regression in the file system watcher subsystem causes iCUE to misinterpret normal read activity as write triggering events.

This produces a repeated loop pattern:

1. CreateFile  
2. QueryOpen  
3. WriteFile  
4. CloseFile  
5. Repeat hundreds of iterations per second  

The sustained write behavior maintains file locks on configuration files, preventing dependent components including the CPUID service from obtaining read access during initialization.

This results in a startup deadlock condition.

---

## Reproduction Steps

1. Install iCUE 5.x  
2. Allow normal system startup  
3. Open Task Manager or Resource Monitor  
4. Observe sustained write activity from iCUE.exe  
5. Restart the CPUID service  
6. Service may hang indefinitely in Starting state  

---

## Mitigation Temporary

- Disable optional logging features  
- Reduce startup modules  
- Monitor disk activity manually  
- Use alternative fan control solutions where applicable  

No official fix is currently available.

---

## Vendor Status

Public user reports indicate ongoing instability across iCUE 5.x releases.  
No formal technical advisory, rollback archive, or remediation timeline has been published at time of writing.
